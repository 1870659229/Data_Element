# 水上航道智能路径规划系统需求文档

## 1. 航道拓扑节点网络提取

期望通过清洗、平滑海量原始 GPS 数据（去除异常漂移点），从真实轨迹中拟合、聚类出现实中的高频航行节点（如真实航道中的拐点、分岔点、汇合点）。以此大幅度降低路径规划时的数据冗余量，构建标准化的水上航道拓扑网络数据集。

## 2. 动态路段耗时权重建模

针对任意两个相邻节点，不单纯依赖物理直线距离，而是基于历史 GPS 轨迹数据（结合速度、方向等特征），拟合出更符合实际水面环境的相邻节点耗时权重，形成高质量的特征标签。

## 3. 路径规划算法设计与改进

依托构建的高质量数据集，设计或改进路径规划算法（如 A*、Dijkstra 的改进版），实现针对特定船舶的个性化导航决策。

### 3.1 特征属性检索

算法需根据目标船舶的身份信息，通过网络查询或关联数据库获取其船长、船宽、船高、载重吨位及满载吃水深度等关键静态特征。

### 3.2 物理约束校验

在规划过程中，必须充分考虑船舶物理限制。例如，大型船舶需规避吃水深度不足的浅滩，高箱船需避开限高不足的桥梁航段。

### 3.3 多目标路径输出

针对给定的起终点，快速输出 2-3 条差异化干预路径（如：安全优先线路、时间最短线路、通航频次最高的综合最优线路），为智能驾驶 UI/UX 界面提供精准决策支持。
