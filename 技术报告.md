## 2. Task 1：数据预处理

### 2.1 做了什么

对原始 AIS 船舶轨迹数据进行清洗、异常去除与轨迹平滑，产出干净的轨迹数据集。输入为 2 个 Excel 文件，输出约 **110.5 万条清洗记录**。

### 2.2 用了什么方法

#### 2.2.1 数据加载与合并

```python
# 读取两个 Excel 文件，按行合并
df = pd.concat([pd.read_excel(fp1), pd.read_excel(fp2)], ignore_index=True)
```

**源文件：** [data_preprocessor.py](file:///e:/Data_Element/data_preprocessor.py#L59-L68)

#### 2.2.2 异常检测（三层过滤）

| 层级 | 方法 | 说明 |
|------|------|------|
| **第1层：基本过滤** | 速度阈值 | 航速范围 [0, 30] 节、时间转换 |
| **第2层：向量化漂移检测** | 逐船计算瞬时速度 | 距离跳变 >500m 或瞬时速度 >30 节 |
| **第3层：IsolationForest 多维异常** | 5维特征联合异常检测 | 航速、计算速度、速度比、航向差、距离 |

**IsolationForest 参数：**
- `contamination=0.01`（预计 1% 异常率）
- `n_estimators=50`
- 特征向量：[reported_speed, calculated_speed, speed_ratio, course_diff, distance]

**源文件：** [data_preprocessor.py](file:///e:/Data_Element/data_preprocessor.py#L105-L199)

#### 2.2.3 轨迹分段

按时间间隔 (>3600s = 1小时) 将每条船的轨迹切分为独立的航行段，过滤点数 <3 的过短段：

```python
time_diffs = group['时间'].diff().dt.total_seconds()
segment_ids = (time_diffs > max_time_gap).cumsum()
```

**源文件：** [data_preprocessor.py](file:///e:/Data_Element/data_preprocessor.py#L218-L247)

#### 2.2.4 卡尔曼滤波平滑

实现一维卡尔曼滤波器，分别对纬度和经度序列进行平滑：

```
预测：p = p + q
更新：k = p / (p + r)
     x = x + k * (measurement - x)
     p = (1 - k) * p
```

**参数配置：**
- `process_noise = 0.01`
- `measurement_noise = 0.1`

**源文件：** [data_preprocessor.py](file:///e:/Data_Element/data_preprocessor.py#L23-L49)（滤波器）、[data_preprocessor.py](file:///e:/Data_Element/data_preprocessor.py#L268-L288)（平滑流程）

#### 2.2.5 地理围栏

限定中国近海区域：纬度 [18°, 42°]，经度 [105°, 125°]。

## 3. Task 2：节点提取

### 3.1 做了什么

从平滑后的轨迹中提取三类关键节点：**拐点**（turn_point）、**途经点**（waypoint）、**停泊点**（stop_point）。共提取 **36,651 个候选节点**（含 8,872 个拐点 + 27,324 个途经点 + 455 个停泊点）。

### 3.2 用了什么方法

#### 3.2.1 Douglas-Peucker 关键点简化

核心思想：递归检测轨迹中偏离首尾连线的最大距离点，保留偏离度 >50m 的关键点，大幅压缩冗余轨迹点。

**算法流程：**
```
1. 连接首尾点 A、B 形成基准线
2. 找到距离基准线最远的中间点 C
3. 若 dist(C, AB) > tolerance → 保留 C，递归处理 AC 和 CB
4. 否则丢弃所有中间点
```

**滑动窗口优化：** 对于 >200 点的长轨迹，采用窗口大小 200、步长 100 的滑动窗口 DP，避免递归深度过大。

**源文件：** [node_extractor.py](file:///e:/Data_Element/node_extractor.py#L176-L248)

#### 3.2.2 曲率计算（拐点识别）

对简化后的关键点，计算局部曲率判断是否为拐点：

```python
# 曲率 = 4 × 三角形面积 / (边长1 × 边长2 × 边长3)
cross = abs(dx1 * dy2 - dy1 * dx2)
area = 0.5 * cross
curvature = 4 * area / (dist1 * dist2 * dist3)
```

拐点判定：曲率 >0.001 **或** 相邻段方向角变化 >30°

**源文件：** [node_extractor.py](file:///e:/Data_Element/node_extractor.py#L250-L273)

#### 3.2.3 停泊点识别

检测连续低速段（航速 <3 节且持续时间 >300s），计算中心位置作为停泊点，并进行网格合并去重。

**源文件：** [node_extractor.py](file:///e:/Data_Element/node_extractor.py#L103-L142)

#### 3.2.4 节点去重与方向特征

在 0.0005° 网格内合并邻近点，计算每个节点的：
- **主导航向**（`heading`）：多艘船在该点航向的圆均值
- **航向集中度**（`heading_concentration`）：反映该节点通行方向的离散程度

**源文件：** [node_extractor.py](file:///e:/Data_Element/node_extractor.py#L275-L321)

#### 3.2.5 节点类型精细分类（KDTree 加速）

对每个节点，用 cKDTree 搜索 100m 范围内的所有轨迹段，分析方向分布：

| 方向聚类数 | 是否有相反方向 | 分类结果 |
|------------|:---:|----------|
| ≥3 个 | 是 | **汇合点** (merge_point) |
| ≥3 个 | 否 | **分岔点** (bifurcation_point) |
| <3 个，圆标准差 >45° | — | **拐点** (turn_point) |
| <3 个，圆标准差 ≤45° | — | **途经点** (waypoint) |

方向聚类使用 DBSCAN(eps=0.5, min_samples=3) 对方向向量 (cosθ, sinθ) 聚类。

**源文件：** [node_extractor.py](file:///e:/Data_Element/node_extractor.py#L388-L474)

## 4. Task 3：节点聚类

### 4.1 做了什么

对上一步的 36,651 个候选节点进行空间聚类，合并空间上相近、航向上一致的节点。最终得到 **2,438 个聚类节点**。

### 4.2 用了什么方法

#### 4.2.1 HDBSCAN 自适应密度聚类（首选）

相比传统 DBSCAN 需要手动指定全局 eps（密度敏感半径），HDBSCAN 能够自适应不同密度区域：

**航向感知特征空间：** 将节点映射到 4 维特征空间：
```
[x_m, y_m, w·sin(θ)·R, w·cos(θ)·R]
```
- `(x_m, y_m)`：经纬度转米坐标
- `(sin(θ)·R, cos(θ)·R)`：航向的加权表示（R 为航向集中度）
- `w = 100`：航向维度权重

**参数：**
- `min_cluster_size = min(max(samples+2, n/100), 5)`
- `cluster_selection_method = 'eom'`（Excess of Mass）

**大数据优化：** 当节点数 >10,000 时，先抽样 5,000 个聚类，再通过 cKDTree 最近邻为全量数据分配标签。

**源文件：** [node_cluster.py](file:///e:/Data_Element/node_cluster.py#L64-L102)

#### 4.2.2 DBSCAN 回退方案

若 HDBSCAN 未安装，回退到 DBSCAN(eps=100m, min_samples=5)，同样使用航向感知特征。

**源文件：** [node_cluster.py](file:///e:/Data_Element/node_cluster.py#L127-L138)

#### 4.2.3 KDE 核密度估计聚类中心

替代简单的加权平均，使用频率加权的 **高斯核密度估计（gaussian_kde）** 寻找密度最高的点作为聚类中心：

```python
kde = gaussian_kde(coords.T, weights=weights / weights.sum())
densities = kde(coords.T)
center = coords[np.argmax(densities)]  # 密度最高的点
```

**源文件：** [node_cluster.py](file:///e:/Data_Element/node_cluster.py#L223-L252)

#### 4.2.4 特殊节点保留

对于被标记为噪声但具有重要航行意义的节点（高频 ≥5 或属于拐点/分岔点/汇合点），保留不丢弃。

**源文件：** [node_cluster.py](file:///e:/Data_Element/node_cluster.py#L254-L280)

#### 4.2.5 小聚类合并优化

将样本量 <3 的小聚类按距离 (150m) 优先合并到同类型的邻近大聚类中，保留重要的小节点。

**源文件：** [node_cluster.py](file:///e:/Data_Element/node_cluster.py#L364-L417)

## 5. Task 4：拓扑网络构建

### 5.1 做了什么

基于聚类后的 2,438 个节点和原始轨迹数据，构建标准化的航道拓扑有向图。最终产出 **1,718 个节点 / 3,388 条边** 的拓扑网络。

### 5.2 用了什么方法

#### 5.2.1 HMM 隐马尔可夫地图匹配

核心创新：不使用简单最近邻，而是采用 HMM 概率化轨迹点→节点匹配。

**HMM 三要素：**

| 要素 | 含义 | 实现 |
|------|------|------|
| **发射概率** | 轨迹点落在某节点的概率 | 高斯分布：`exp(-距离² / 2·100²)` |
| **转移概率** | 从一个节点转移到另一个的概率 | 节点间距离与轨迹实际距离的一致性：`exp(-|diff| / max(actual, 100))` |
| **初始概率** | 起点在各节点的概率 | 均匀 = 发射概率 |

**源文件：** [topology_builder.py](file:///e:/Data_Element/topology_builder.py#L154-L213)

#### 5.2.2 Viterbi 解码

动态规划求最大概率节点序列：

```
dp[t][j] = max( dp[t-1][i] × trans_prob(i→j) × emission_prob(j) )
```

解码后去除连续相同节点的冗余。

**源文件：** [topology_builder.py](file:///e:/Data_Element/topology_builder.py#L215-L308)

#### 5.2.3 边属性统计

对每条边聚合统计：

| 属性 | 含义 |
|------|------|
| `count` | 通过次数 |
| `ships` | 通过的船舶集合 |
| `avg_distance` | 平均物理距离 |
| `avg_time` | 平均通行耗时 |
| `avg_speed` | 平均航速 |
| `shape_points` | 航道曲线形状点 |
| `hourly_counts` | 24 小时时段分布 |
| `direction_counts` | 正向/反向通行次数 |
| `is_bidirectional` | 是否为双向航道 |
| `predicted_times` | 24 小时预测耗时 |

**源文件：** [topology_builder.py](file:///e:/Data_Element/topology_builder.py#L59-L152)

#### 5.2.4 时间依赖图

为每条边计算 24 小时时间依赖权重，基于时段可靠性因子：

```
reliability(h) = min(1.0, hour_freq(h) / (total_freq / 24))
predicted_time(h) = avg_time × (1.0 - reliability × 0.2)
```

#### 5.2.5 低权重边过滤与节点合并

- 删除权重 <3 的低频边及孤立节点
- 合并距离 <50m 的相似节点，保留频率最高的为主节点

**网络统计：**
- 平均聚类系数
- 连通分量数
- 最大连通分量大小
- 平均度

## 6. Task 5：动态路段耗时权重建模

### 6.1 做了什么

基于拓扑网络和轨迹数据，构建机器学习模型预测每条边在每个时段的动态耗时权重。对比 **5 种模型**，自动选择最优。

### 6.2 用了什么方法

#### 6.2.1 核心创新：time_ratio 目标变量

传统做法直接预测 travel_time，但受距离主效应支配导致模型学习困难。

**本系统创新：** 定义 `time_ratio = actual_time / theoretical_time`

- `theoretical_time = avg_distance / (avg_reported_speed × 0.5144)`
- 归一化掉距离/速度主效应，让模型专注学习动态偏差
- 预测时将 `ratio × theoretical_time` 还原为实际耗时

**源文件：** [advanced_weight_model.py](file:///e:/Data_Element/advanced_weight_model.py#L521-L647)

#### 6.2.2 22 维特征工程

| 类别 | 特征（共 22 维） | 说明 |
|------|------------------|------|
| **速度特征** | avg_reported_speed, std_reported_speed, speed_cv | 报告航速统计量 |
| **方向特征** | bearing, bearing_sin, bearing_cos, avg_course_change, std_course_change | 航向的正弦/余弦表示 |
| **时段特征** | hour, hour_sin, hour_cos, is_peak_hour, is_weekend | 时间周期性编码 |
| **水域特征** | waterway_type, course_change_x_narrow | 窄/宽水域×交互 |
| **网络特征** | node_degree_from, node_degree_to, edge_betweenness | 图拓扑统计 |
| **统计特征** | sample_count, log_sample_count | 样本量及对数 |
| **交互特征** | narrow_x_peak, course_change_x_peak | 水域×时段交互 |

**注意：** 特意移除了 `avg_travel_time` 和 `avg_actual_speed` 等泄漏特征。

**源文件：** [advanced_weight_model.py](file:///e:/Data_Element/advanced_weight_model.py#L168-L177)

#### 6.2.3 5 模型对比

| 模型 | 算法 | 超参搜索 |
|------|------|----------|
| **XGBoost** | 梯度提升树 | Optuna 贝叶斯 20 trials |
| **LightGBM** | 轻量级梯度提升 | Optuna 贝叶斯 30 trials |
| **LightGBM-Tweedie** | Tweedie 回归（处理正偏分布） | Optuna 贝叶斯 30 trials |
| **Random Forest** | 随机森林 | Optuna 贝叶斯 20 trials |
| **MLP** | 多层感知机（PyTorch） | 5-fold CV 网格搜索 |
| **GNN ★** | 图注意力网络（GAT + EdgeMLP） | 4组参数网格搜索 |

#### 6.2.4 GNN 模型架构

```
GATConv(node_dim → hidden_dim, heads=2) × N层
    ↓ BatchNorm + ReLU + Dropout
EdgeMLP:
    concat[h_u, h_v, edge_attr]
    → Linear(128) → BN → ReLU
    → Linear(64)  → BN → ReLU
    → Linear(1)
```

**训练配置：** HuberLoss(δ=2.0), AdamW, ReduceLROnPlateau, 早停 patience=50

**源文件：** [advanced_weight_model.py](file:///e:/Data_Element/advanced_weight_model.py#L79-L113)（网络定义）、[advanced_weight_model.py](file:///e:/Data_Element/advanced_weight_model.py#L1195-L1431)（训练流程）

#### 6.2.5 混合策略：经验统计 + 模型插值

预测阶段采用智能混合：

| 数据密度 | 策略 |
|----------|------|
| 密集时段（≥2 个轨迹样本） | **直接用经验均值**（最可靠） |
| 稀疏时段（<2 个样本） | **模型预测插值** |
| 完全无数据边 | **模型外推**（仅用结构特征） |

这样既保留了有数据时的统计精度，又通过模型填补了稀疏和无数据的情况。

**源文件：** [advanced_weight_model.py](file:///e:/Data_Element/advanced_weight_model.py#L1530-L1899)

#### 6.2.6 模型持久化

完整的模型保存/加载机制（`save_model` / `load_model`），包含：
- 最优模型对象
- StandardScaler
- 密度网格与阈值
- 网络特征（度、介数中心性）
- 水域类型映射
- 特征重要性

## 7. Task 7：船舶个性化导航决策

### 7.1 做了什么

根据目标船型和起终点，进行个性化多目标路径规划，输出差异化导航方案。

由 4 个子模块组成：
1. **ShipCharacteristicsManager** — 船舶特征检索
2. **PhysicalConstraintChecker** — 物理约束校验
3. **MultiObjectiveNavigator** — 多目标路径规划
4. **NavigationDecisionMaker** — 导航决策输出

**源文件：** [ship_navigator.py](file:///e:/Data_Element/ship_navigator.py)

### 7.2 ShipCharacteristicsManager（特征检索）

#### 做了什么

根据船舶身份信息获取：船长、船宽、船高、载重吨位、满载吃水深度。

#### 用了什么方法

**三层检索机制：**

| 层级 | 方法 | 说明 |
|------|------|------|
| 第1层 | **关键词匹配** | 船名含"集装箱"→集装箱船，含"油轮"→油轮 |

### 7.3 3.2 PhysicalConstraintChecker（物理约束）

#### 做了什么

在规划过程中充分考虑船舶物理限制：
- 大型船舶规避吃水深度不足的浅滩
- 高箱船避开限高不足的桥梁航段
- 宽体船限制通过窄水道

#### 用了什么方法

**三级约束体系：**

| 约束类型 | 窄水道+短距离 | 窄水道 | 开阔海域 |
|----------|:---:|:---:|:---:|
| 最小水深 | 6.0m | 8.0m | 15.0m |
| 最大宽度 | 40.0m | 60.0m | 120.0m |
| 最大限高 | 20.0m | 30.0m | 60.0m |

**通过性判断：** 20% 安全裕度
```python
passable = (ship_draft <= min_depth × 1.2) and
           (ship_height <= max_height × 1.2) and
           (ship_width <= max_width × 1.2)
```

**ML 增强：**
- **RiskPredictionModel**（GradientBoostingRegressor）：18 维特征 → 0-100 风险评分
- **PassabilityProbabilityModel**（GradientBoostingClassifier）：14 维特征 → 0-1 通过概率
- **MultiTaskDNN**（PyTorch）：SiLU+LayerNorm+NLL高斯损失 → 风险+可达性联合预测

**源文件：** [ship_navigator.py](file:///e:/Data_Element/ship_navigator.py#L340-L539)、[navigation_models.py](file:///e:/Data_Element/navigation_models.py)

### 7.4 MultiObjectiveNavigator（路径规划）

#### 做了什么

针对给定起终点，快速输出多目标差异化路径。

#### 用了什么方法

**核心算法：** NetworkX Dijkstra 最短路径，配合不同边权重函数实现多目标：

| 路径类型 | 权重策略 | Dijkstra cost |
|----------|----------|:---:|
| **距离最短** | 物理距离 | `haversine_distance / 1000` |
| **时间最短** | 动态耗时 | `avg_distance / (avg_speed × 1.852)` |
| **综合最优** | 多因素排序 | 优先低阻塞、多边数路径 |

**备选路径生成：** 当主路径不足 3 条时，逐一移除主路径上的一条边（Edge Deletion），重新运行 Dijkstra 生成差异化备选。

**源文件：** [app.py](file:///e:/Data_Element/app.py#L137-L318)（Web API 的路径规划逻辑）

### 7.5 路径类型的完整策略矩阵

| 类型 | 安全权重 | 时间权重 | 距离权重 | 频次权重 | 策略描述 |
|------|:---:|:---:|:---:|:---:|------|
| **安全优先** | 60% | 10% | 15% | 15% | 规避高风险航段，优先宽阔水道 |
| **时间最短** | 10% | 60% | 15% | 15% | 动态耗时最小，接受可控风险 |
| **距离最短** | 10% | 10% | 60% | 20% | 物理距离最短路径 |
| **通航频次最高** | 15% | 10% | 15% | 60% | 高频通行航段，群体智慧 |
| **综合最优** | 35% | 25% | 20% | 20% | 四维加权平衡 |
